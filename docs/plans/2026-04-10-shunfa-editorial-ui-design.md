# 顺发小程序编辑部视觉设计

**目标**

把当前偏模板化的小程序界面，改成更有记忆点的“编辑部晨报感”：像每天早上在一张纸质选题单上选稿、起稿、签发，而不是普通效率工具。

**设计方向**

- 基底采用偏米白纸感背景，替代纯白和冷灰。
- 绿色只作为“签发 / 可发布 / 进度推进”的高亮色，不再大面积铺满。
- 增加荧光黄便签、细描边卡片、编辑标注条等元素，建立统一视觉语言。
- 组件保持信息清晰，但弱化“面板仪表盘”味道，强化“稿件、档案、发稿台”的隐喻。

**页面定位**

- 首页：今日发稿台。强调今日状态、连更、等级进度和“开始写作”的主动作。
- 选题页：晨报选题单。每条热点像待签发稿件，突出来源、摘要和原文链接保留。
- 预览页：待发稿样张。主卡片像排版稿纸，事实区像编辑备注。
- 我的页：个人写作档案。强调累计记录、近期状态、提醒入口。

**实现范围**

- 改造 `app.wxss` 里的全局 token 和共享卡片 / 按钮风格。
- 重做首页、选题页、预览页、我的页的结构和样式。
- 调整热点卡片、连更、钻石和等级组件，使其与新视觉一致。
- 不改动核心业务流程，不新增页面，不改变现有 API 交互。

**视觉规则**

- 使用高对比正文色和细边框，减少大面积阴影。
- 用少量层叠、便签、角标制造节奏，不做花哨渐变。
- 动效以轻微上浮、淡入、压感反馈为主，不使用浮夸动画。

**验收标准**

- 首页、选题页、预览页、我的页在视觉上是一套统一语言。
- 热点、事实来源、原文链接和发稿动作的主次层级更清晰。
- 页面在微信开发者工具里能正常显示，关键 CTA 不受样式改造影响。
