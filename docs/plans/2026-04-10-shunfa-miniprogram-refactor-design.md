# 顺发小程序重构设计

**目标**

把顺发从 Coze 末端大模型输出迁移为“小程序直连自有后端”的结构，保证热点数据和原始链接以结构化 JSON 原样返回，避免 Coze/豆包对最终消息的改写。

**当前约束**

- 当前 workspace 中未找到 `product-shunfa-prd.md`、`research/shunfa_skill_v2.md`、`research/judgment_calibrator.md`
- 设计基于现有 FastAPI、SQLite、小程序代码和用户明确需求展开

**核心判断**

1. 热点链路必须彻底脱离 LLM。
   RSS 抓取、解析、去重、排序、入库、返回 JSON 全部走确定性逻辑，只在“内容生成”阶段调用 DeepSeek。

2. MVP 不需要多轮讨论。
   先做“今日 3 条热点 -> 选 1 条 -> 一键生成 -> 预览确认 -> 发布记录 -> 连更追踪”，用最短路径验证产品价值。

3. 保留现有 FastAPI + SQLite 主干。
   不推翻现有用户、checkin、提醒设置模型，只补一个本地热点表和对应 API。

**系统结构**

- 微信小程序
  - 登录
  - 热点列表
  - 一键生成
  - 预览确认
  - 连更与提醒设置
- FastAPI
  - 登录鉴权
  - 热点聚合 API
  - 内容生成 API
  - 发布确认 API
  - 提醒设置 API
  - 微信订阅消息发送服务
- SQLite
  - users
  - checkins
  - topic_history
  - achievements
  - hot_topics

**数据流**

1. 定时任务抓取 RSS 源。
2. 解析出标题、摘要、原始 URL、发布时间、来源。
3. 用确定性规则去重和筛选，AI 只补“正向角度/反角度”，不改原始热点字段。
4. 热点写入 SQLite `hot_topics` 表。
5. 小程序请求当天热点接口，直接拿结构化 JSON。
6. 用户选题后创建/重置当日 checkin。
7. 调用内容生成接口，生成初稿并写回 checkin。
8. 预览页确认后发布，更新 streak/points。
9. 提醒模块在用户授权后按提醒时间发送微信订阅消息。

**MVP 范围**

v1 包含：

- 微信登录
- 今日 3 条热点
- 原始链接可见
- 一键生成内容
- 发布确认
- 连更追踪
- 提醒设置

v1 不包含：

- 深度讨论模式
- 复活机制
- 成就系统大改
- 多平台发文
- 管理后台

**微信推送方案**

推荐使用“小程序订阅消息”：

- 前端调用 `wx.requestSubscribeMessage`
- 后端保存提醒时间
- 定时任务扫描到点未完成用户
- 调用微信订阅消息发送接口
- 点击消息跳回小程序热点页/预览页

限制：

- 用户必须授权
- 需要模板 ID
- 适合“每日一次提醒”，不适合高频推送

**实施策略**

第一阶段先做“热点结构化 + 一键生成 + 发布确认”全链路。
第二阶段再补“微信订阅消息发送”和定时提醒。
